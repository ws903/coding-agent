# ADR 0001 — qwen3.6:35b as the default model

* Status: Accepted
* Date: 2026-05-22

## Context

The agent needs a strong-enough model that fits on a single 16 GB consumer GPU (the project's reference hardware: NVIDIA RTX 5070 Ti). Until 2026-05, the default was `qwen3:14b` (dense, ~9 GB, SWE-bench Verified ~45). The local-model menu has broadened — what's the best fit today?

## Decision

Default model is **`qwen3.6:35b`** (MoE, 35B total / ~3B active per token, ~16 GB Q3 / 22 GB Q4 with RAM offload, SWE-bench Verified 73.4).

Set via `LLMClient.__init__(model="qwen3.6:35b")` and `cli.py:DEFAULT_MODEL`. Overridable per-call via `--model` or `AGENT_MODEL` env var.

## Rationale

- **Quality jump.** SWE-bench Verified ~45 → 73.4 vs the previous `qwen3:14b` default — almost a generation leap for agentic tasks.
- **MoE architecture fits the hardware.** Only ~3B parameters are active per token, so the inactive expert weights can live in DDR5 RAM with negligible throughput cost (measured ~101 tok/s on a 3090 in the same RAM-offload config).
- **Native tool calling reliable.** Verified live that qwen3.6:35b emits valid OpenAI-format `tool_calls` consistently; `chat_with_tools` works without parsing workarounds.
- **256K context window.** Way over what `qwen3:14b` offered (40K). Matters once the codebase symbol map + cross-step context start filling prompts.
- **Multimodal.** The model accepts images. Currently unused but lets future work (screenshot tools, vision skills) land without a model change.

## Consequences

- **VRAM is tighter.** Q3_K_M needs ~16.6 GB on the GPU + KV cache; Q4_K_M needs ~22 GB total with offload. Requires 32 GB+ system RAM for the offload path. Documented in README.
- **Thinking mode is on by default.** This model spends real time in `delta.reasoning` before emitting `delta.content`. Two related decisions cope with it: streaming reasoning (ADR 0003 implications) and `quick_chat_stream` with `think:false` for the fast-chat path (ADR 0006).
- **`qwen3:14b` stays documented as the small-VRAM fallback** for users without 32 GB system RAM.

## Alternatives considered

- **`qwen3-coder:30b`** — 19 GB MoE, code-specialized. Skipped: qwen3.6 supersedes it (newer, better SWE-bench, multimodal).
- **`qwen3.6:27b` dense** — 17 GB, no offload path. Borderline VRAM fit; would limit context. Documented as a fallback for users who don't want offload.
- **Splitting planner/executor across specialist models** — Adds 5-15s of model-swap latency per phase transition on 16 GB VRAM and qwen3.6's internal MoE router already specializes per token. See `feedback_no_router_for_now.md` notes.
